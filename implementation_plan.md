# Сервис Бронирования Переговорки через Telegram

В этом документе описывается технический подход к созданию премиального веб-сервиса для бронирования переговорной комнаты, доступ к которому имеют только участники определенной группы в Telegram.

## Архитектура и стек технологий

- **Фреймворк**:Fat api (App Router, TypeScript)
- **Стилизация**: Tailwind CSS + Framer Motion (неординарный, оригинальный стиль "Google Antigravity" — левитирующие предметы, сложная физика, необычные сетки и глубокий контраст)
- **База данных**:  PostgreSQL
- **Аутентификация**: Telegram Authentication API
- **Календарь**: Полностью кастомная анимированная сетка в стиле Antigravity
- **Уведомления**: Telegram Bot API 

## Предлагаемые изменения

### Схема базы данных (Prisma)
- **User (Пользователь)**: Telegram ID, Имя, Username, Роль.
- **Booking (Бронь)**: ID, Название встречи, Время начала, Время окончания, UserId (связь с пользователем).

### Основные компоненты
#### `/app/api/auth/[...nextauth]`
Интеграция с входом через Telegram. Шаг проверки должен вызывать метод `getChatMember` через Telegram Bot API, чтобы убедиться, что пользователь состоит в нужной группе.

#### `/app/api/bookings`
REST endpoints для получения списка, создания и удаления бронирований. Должна быть проверка на пересечение времени бронирования.

#### `/components/Calendar`
Визуальное отображение расписания переговорной комнаты. Удобный интерфейс для выбора свободного времени.

#### Интеграция с Telegram Ботом
Фоновая утилита для отправки сообщений в групповой чат или конкретному пользователю при создании или отмене бронирования.

## План проверки (Верификация)

### Автоматизированное тестирование
- Валидация логики пересекающихся бронирований.
- Убедиться, что неавторизованные пользователи (не из группы) получают отказ в доступе.

### Ручная проверка
- Локальное тестирование пользовательского интерфейса на предмет адаптивности, анимаций и эстетики.
- Симуляция процесса входа через Telegram и создания бронирования для проверки уведомлений от бота.
